Else

Bestimmt einen oder mehrere Befehle, die ausgeführt werden sollen, wenn eine IF-Anweisung als FALSE (falsch) gewertet wird. Wenn mehr als ein Befehl vorhanden ist, müssen sie mit einem Block (geschweifte Klammern) umschlossen werden.

Else

Bemerkungen

Jedes ELSE muss mit einer darüberliegenden IF-Anweisung verbunden werden. Ein ELSE bezieht sich immer auf die nächst darüberliegende noch nicht beanspruchte IF-Anweisung, sofern dieses Verhalten nicht mit einem Block geändert wurde.

Unmittelbar nach einem ELSE kann ein einzelner beliebiger Befehl erfolgen. Diese Methode wird häufig bei "else if"-Konstruktionen verwendet (siehe Beispiele unten).

Hat ein IF oder ELSE mehr als eine Zeile, müssen solche Zeilen von geschweiften Klammern umschlossen werden. Hat ein IF oder ELSE allerdings nur eine Zeile, können die geschweiften Klammern weggelassen werden. Zum Beispiel:

if Anzahl > 0  ; Geschweifte Klammern können unten weggelassen werden, weil es nur eine Zeile ist.
    MsgBox Drücken Sie OK`, um den Ablauf zu beginnen.
else  ; Geschweifte Klammern sind unten notwendig, weil mehrere Zeilen vorhanden sind.
{
    WinClose Unbenannt - Editor
    MsgBox Es sind keine Elemente vorhanden.
}

Der One True Brace Style (OTB) kann je nach Bedarf bei einem "else" angewendet werden. Zum Beispiel:

if IstFertig {
    ...
} else if (x < y) {
    ...
} else {
    ...
}

Siehe auch

Siehe Blöcke. Jeder IF-Befehl kann ein ELSE verwenden, einschließlich IfWinActive, IfWinExist, IfMsgBox, IfInString, IfBetween, IfIn, IF und IF (Ausdruck).

Beispiele

IfWinExist, Unbenannt - Editor
{
    WinActivate
    Send Das ist ein Test.{Enter}
}
else
{
    WinActivate, Irgendein anderes Fenster
    MouseClick, left, 100, 200
}

if x = 1
    Gosub, a1
else if x = 2 ; "else if"-Stil
    Gosub, a2
else IfEqual, x, 3 ; alternativer Stil
{
    Gosub, a3
    Sleep, 1
}
else Gosub, a4  ; Einzelne Befehle können unmittelbar nach ELSE erfolgen.

; Auch OK:
IfEqual, y, 1, Gosub, b1
else {
    Sleep, 1
    Gosub, b2
}
