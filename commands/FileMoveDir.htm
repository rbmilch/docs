FileMoveDir

Verschiebt einen Ordner, einschließlich seine Unterordner und Dateien. Kann einen Ordner auch umbenennen.

FileMoveDir, Quelle, Ziel , Flag

Parameter

Quelle

Name des Quellverzeichnisses (ohne umgekehrten Schrägstrich danach), das im A_WorkingDir-Verzeichnis vermutet wird, wenn kein absoluter Pfad angegeben ist. Zum Beispiel: C:\Mein Ordner

Ziel

Der neue Pfad und Name des Verzeichnisses (ohne Backslash), das im A_WorkingDir-Verzeichnis vermutet wird, wenn kein absoluter Pfad angegeben ist. Zum Beispiel: D:\Mein Ordner.

Hinweis: Ziel ist der aktuelle Pfad und Name, die das Verzeichnis haben wird, nachdem es verschoben wurde; es ist nicht das Verzeichnis, in dem Quelle verschoben worden ist (außer bei der unten erwähnten Einschränkung).

Flag

(Optionen) Geben Sie eines der folgenden Zeichen an:

0 (Standard): Vorhandene Dateien werden nicht überschrieben. Die Operation wird fehlschlagen, wenn Ziel bereits als Datei oder Verzeichnis existiert.

1: Vorhandene Dateien werden überschrieben. Allerdings werden Dateien oder Unterordner innerhalb von Ziel, die kein Gegenstück in Quelle haben, nicht gelöscht. Bekannte Einschränkung: Wenn Ziel bereits als Ordner vorhanden ist und befindet es sich auf den gleichen Datenträger wie Quelle, wird Quelle auf das Ziel verschoben, anstatt es zu überschreiben. Diese Einschränkung kann mit der nächsten Option umgangen werden.

2: Das gleiche wie Modus 1, außer dass die Einschränkung fehlt.

R: Benennt das Verzeichnis um, anstatt es zu verschieben. Zwar ist Umbenennen prinzipiell das gleiche wie Verschieben, allerdings könnte es dort nützlich sein, wo "Alles oder nichts" gelten soll; das heißt, um zu verhindern, dass die Operation nur teilweise erfolgreich ist, wenn Quelle oder eine ihrer Dateien gesperrt ist (in Benutzung). Diese Methode kann Quelle nicht auf einen anderen Datenträger verschieben, sondern nur auf den gleichen Datenträger in ein beliebig anderen Verzeichnis. Die Operation wird fehlschlagen, wenn Ziel bereits als Datei oder Verzeichnis vorkommt.

ErrorLevel

[v1.1.04+]: Dieser Befehl ist in der Lage, bei Misserfolg eine Ausnahme auszulösen. Weitere Informationen finden Sie unter Laufzeitfehler.

ErrorLevel wird bei Misserfolg auf 1 und bei Erfolg auf 0 gesetzt.

Bemerkungen

FileMoveDir verschiebt nur einen einzigen Ordner auf eine neue Position. In dem Beispielabschnitt von FileMove wird gezeigt, wie die Inhalte eines Ordners (alle Dateien und Unterordner) verschoben werden können.

Befinden sich Quelle und Ziel auf unterschiedlichen Datenträgern oder UNC-Pfade, wird eine Kopieren/Löschen-Operation erfolgen, anstatt Verschieben.

Siehe auch

FileCopyDir, FileCopy, FileMove, FileDelete, Dateimuster-Schleife, FileSelectFolder, SplitPath

Beispiel

FileMoveDir, C:\Mein Ordner, D:\Mein Ordner  ; Verschiebt den Ordner auf ein neues Laufwerk.
FileMoveDir, C:\Mein Ordner, C:\Mein Ordner (umbenannt), R  ; Benennt den Ordner einfach um.
FileMoveDir, C:\Mein Ordner, C:\Neuer Standort\Mein Ordner, R  ; Ordner können per Umbenennen auf beliebige Positionen verschoben werden, solange sie sich auf demselben Datenträger befinden.
