GroupDeactivate

Ähnlich wie GroupActivate, außer dass das nächste Fenster aktiviert wird, welches nicht zur Gruppe gehört.

GroupDeactivate, Gruppenname , R

Parameter

Gruppenname

Name der Gruppe, die ursprünglich mit GroupAdd definiert worden ist.

R

Legt fest, ob das älteste oder das neueste Fenster aktiviert werden soll, wenn kein einziges Fenster aus der Gruppe aktiv ist. Fehlt dieser Parameter, wird immer das älteste Fenster aktiviert. Ist der Buchstabe R angegeben, wird das neueste (zuletzt aktive) Fenster aktiviert, aber nur, wenn beim Ausführen des Befehls kein Fenster der Gruppe aktiv ist. "R" ist beispielsweise nützlich, wenn Sie vorübergehend eine andere Aufgabe durchführen wollen. Wenn Sie dann mit GroupActivate, GroupDeactivate oder GroupClose zur Gruppe zurückkehren, wird nicht das älteste, sondern das aktuell genutzte Fenster aktiviert.

Bemerkungen

Dieser Befehl bewirkt, dass das erste Fenster, das nicht mit den Spezifikationen der Fenstergruppe übereinstimmt, aktiviert wird. Verwendet man den Befehl danach nochmals, wird das nächste Fenster in der Reihe aktiviert und so weiter. Normalerweise weist man diesen Befehl einem Hotkey zu, so dass diese Fensterdurchschaltung durch Drücken dieser Taste automatisiert werden kann.

Dieser Befehl ist besonders dann nützlich, wenn man mehrere bevorzugte Fenster hat, die nahezu immer laufen. Durch Hinzufügen dieser Fenster zu einer Gruppe können Sie mit GroupDeactivate jedes Fenster besuchen, das nicht zu Ihren Favoriten gehört, und dann entscheiden, ob Sie dieses Fenster schließen wollen oder nicht. Dadurch können Sie Ihr Desktop sehr viel schneller aufräumen, als es manuell möglich wäre.

Siehe GroupAdd für mehr Details über Fenstergruppen.
