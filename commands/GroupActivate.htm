GroupActivate

Aktiviert das nächste Fenster in einer Fenstergruppe, die mit GroupAdd definiert worden ist. 

GroupActivate, Gruppenname , R

Parameter

Gruppenname

Name der zu aktivierenden Gruppe, die ursprünglich mit GroupAdd definiert worden ist.

R

Legt fest, ob das älteste oder das neueste Fenster aktiviert werden soll, wenn kein einziges Fenster aus der Gruppe aktiv ist. Fehlt dieser Parameter, wird immer das älteste Fenster aktiviert. Ist der Buchstabe R angegeben, wird das neueste (zuletzt aktive) Fenster aktiviert, aber nur, wenn beim Ausführen des Befehls kein Fenster aus der Gruppe aktiv ist. "R" ist beispielsweise nützlich, wenn Sie vorübergehend eine andere Aufgabe durchführen wollen. Wenn Sie dann mit GroupActivate, GroupDeactivate oder GroupClose zur Gruppe zurückkehren, wird nicht das älteste, sondern das aktuell genutzte Fenster aktiviert.

ErrorLevel [AHK_L 54+]

ErrorLevel wird auf 1 gesetzt, wenn kein Fenster zum Aktivieren gefunden wurde, ansonsten auf 0.

Bemerkungen

Dieser Befehl bewirkt, dass das erste Fenster, das mit den Spezifikationen der Fenstergruppe übereinstimmt, aktiviert wird. Verwendet man den Befehl danach nochmals, wird das nächste Fenster in der Reihe aktiviert und so weiter. Normalerweise weist man diesen Befehl einem Hotkey zu, so dass diese Fensterdurchschaltung durch Drücken dieser Taste automatisiert werden kann.

Wenn ein Fenster unmittelbar nach Aktivieren eines anderen Fensters aktiviert wird, kann es passieren, dass die Buttons der Taskleiste anfangen zu blinken (abhängig vom Betriebssystem und von den Einstellungen). Mit #WinActivateForce kann das verhindert werden.

Siehe GroupAdd für mehr Details über Fenstergruppen.
