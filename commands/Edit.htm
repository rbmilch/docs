Öffnet das aktuelle Skript mit dem zugeordneten Editor.

Edit

Der Edit-Befehl öffnet das aktuelle Skript mithilfe des zugeordneten Verbs "edit", das in der Registry angegeben ist (oder mit dem Windows-Texteditor, wenn kein Verb vorhanden ist). Stellt sich heraus, dass ein Editorfenster das Skript bereits offen hat (abhängig vom Fenstertitel), wird dieses Fenster aktiviert, anstatt eine neue Instanz des Editors zu öffnen.

Dieser Befehl hat keine Wirkung, wenn er innerhalb eines kompilierten Skripts ausgeführt wird.

Verschiedene Texteditoren unterstützen eine Syntaxhervorhebung für AutoHotkey (siehe unten). Desweiteren kann eine kontextabhängige Hilfe für AutoHotkey-Befehle mit diesem Beispiel aktiviert werden. Um die eigene Produktivität zu erhöhen, kann ein Auto-Vervollständigungs-Tool wie ISense genutzt werden, das bei fast jedem Editor funktioniert. Es überwacht, was Sie eingeben, und zeigt Menüs und Parameterlisten an, die Ihnen einen Teil der Eingabe abnimmt und Sie an die Reihenfolge der Parameter erinnert.

Beispiel

Edit ; Öffnet das Skript zum Editieren.
; Wenn die Befehlszeilenstruktur Ihres Editors wie folgt aufgebaut ist,
; kann dieses Skript genutzt werden, um ihn als Standardeditor für ahk-Dateien festzulegen:
;
;   Editor.exe "Vollständiger Pfad des Skripts.ahk"
;
; Wenn Sie das Skript ausführen, werden Sie aufgefordert,
; die ausführbare Datei Ihres Editors auszuwählen.
;
FileSelectFile Editor, 2,, Wählen Sie Ihren Editor aus, Programme (*.exe)
if ErrorLevel
    ExitApp
RegWrite REG_SZ, HKCR, AutoHotkeyScript\Shell\Edit\Command,, "%Editor%" "`%1"

Editoren mit AutoHotkey-Unterstützung

SciTE4AutoHotkey ist eine benutzerdefinierte Version des Texteditors SciTE, der speziell zum Editieren von AutoHotkey-Skripten entwickelt wurde. Seine Features umfassen:

  • Syntax-Hervorhebung
  • Intelligente automatische Einrückung
  • Automatische Vervollständigung
  • Aufruftipps (auch als IntelliSense bekannt)
  • Code-Folding
  • Unterstützung für interaktive Debuggen
  • Weitere Tools für AutoHotkey-Skripte

SciTE4AutoHotkey kann hier gefunden werden: http://fincs.ahk4.net/scite4ahk/

AHK Studio ist ein Editor für AutoHotkey-Skripte, der mit AutoHotkey erstellt wurde. Im folgenden Forum-Thread finden Sie genaue Details, Video-Anleitungen und eine ständig wachsende Liste von Features: AHK Studio

AutoGUI ist eine integrierte Entwicklungsumgebung für AutoHotkey, die einen GUI-Designer mit einem Skripteditor verknüpft. Sie finden es hier: AutoGUI - GUI Designer and Script Editor

Weitere Editoren, für die die Syntaxhervorhebung für AutoHotkey aktiviert werden kann:

  • AkelPad
  • Crimson Editor
  • Eclipse, FAR-Manager, und andere Editoren, die Colorer take5 verwenden
  • Emacs
  • Notepad++
  • Notepad2
  • PSPad
  • SciTE und möglicherweise andere auf Scintilla basierende Editoren
  • Sublime Text Editor
  • Total Commander mit Synplus-Plugin

Zusätzlich enthält die Zip-Version von AutoHotkey Basic (http://www.autohotkey.com/download/1.0/) Dateien zum Aktivieren der Syntaxhervorhebung für folgende Editoren. Allerdings sind einige dieser Dateien veraltet und funktionieren möglicherweise nicht mehr:

  • ConTEXT
  • EditPlus
  • EmEditor
  • jEdit
  • MED
  • TextPad
  • UltraEdit
  • Vim

Wenn Ihr Texteditor hier nicht aufgelistet ist, werden Sie in den Foren vielleicht mehr Glück haben.
